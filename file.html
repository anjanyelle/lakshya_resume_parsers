1

Prepare the Machine

PHASE 01 / 06
Before installing anything, make sure your hardware/VM meets the specs above.

💿Install Windows OS

Install Windows Server 2016 (recommended) or Windows 10. Ensure it is activated with a valid license.

  • Set language to English (United States)
  • Select "Windows Server 2016 Standard (Desktop Experience)"
  • Assign the entire disk to C:\ during installation
  • Set a strong Administrator password

💾Create Dedicated D:\ Drive

The application requires a dedicated D:\ drive. If you have one physical disk, create a partition.

# Open Disk Management diskmgmt.msc # Right-click C: → Shrink Volume # Shrink by at least 60,000 MB (60 GB) # Right-click unallocated → New Simple Volume # Assign drive letter: D # Format as NTFS, Label: "AppData"

🌐Set Static IP Address (LAN)

Assign a fixed internal IP so office computers can always find the server.

# Open Network Settings → Ethernet → Properties # Select: Internet Protocol Version 4 (TCP/IPv4) # Choose: Use the following IP address IP Address : 192.168.1.100 Subnet Mask : 255.255.255.0 Gateway : 192.168.1.1 (your router IP) DNS Server : 8.8.8.8 (Google DNS)
2

Install Required Software

PHASE 02 / 06

⚙️Install .NET Framework 4.5.2 or Higher

Open PowerShell as Administrator and run:

# Check current version first Get-ChildItem 'HKLM:\SOFTWARE\Microsoft\NET Framework Setup\NDP' -Recurse | Get-ItemProperty -Name version -EA 0 | Where { $_.PSChildName -Match '^(?!S)\p{L}'} | Select PSChildName, version # If version is below 4.5.2, download from: # https://www.microsoft.com/en-us/download/details.aspx?id=42643 # Run the installer and restart the server

🖥️Enable IIS (Internet Information Services)

IIS is the web server that will host the application and allow browser access from other PCs.

# Open Server Manager → Add Roles and Features # Role: Web Server (IIS) # Enable these features: ✔ Web Server ✔ Common HTTP Features ✔ Default Document ✔ Static Content ✔ HTTP Errors ✔ Application Development ✔ ASP.NET 4.5 ✔ .NET Extensibility 4.5 ✔ ISAPI Extensions ✔ ISAPI Filters ✔ Health and Diagnostics ✔ HTTP Logging ✔ Security ✔ Windows Authentication ✔ Request Filtering ✔ Management Tools ✔ IIS Management Console (IIS Manager)
ℹ️
After installation, open your browser and go to http://localhost — you should see the IIS Welcome page. This confirms IIS is running.

🔧Install IIS Manager (version 8+)

IIS Manager is used to configure websites, ports, SSL certificates, and application pools.

# IIS Manager is installed with IIS above. # To verify version — open IIS Manager: inetmgr # Help → About → confirm version 8.x or higher

🌍Install Google Chrome + Extension

Install Chrome on every recruiter's PC (not the server). Then load the unpacked Chrome extension provided by CEIPAL/Naukri.

  • Open Chrome → go to chrome://extensions/
  • Enable "Developer Mode" (top-right toggle)
  • Click "Load Unpacked" → browse to the extension folder
  • Confirm the extension appears and is enabled
3

Configure IIS & Application

PHASE 03 / 06

📁Deploy Application Files to D:\ Drive

Copy all application files to the D:\ drive as instructed by your CEIPAL setup package.

# Example folder structure: D:\CEIPALApp\ ├── bin\ ├── config\ ├── web.config └── index.html

🔗Create a New IIS Website

In IIS Manager, create a new site pointing to your application folder.

# Open IIS Manager → inetmgr # Right-click "Sites" → Add Website Site Name : CEIPALApp Physical Path: D:\CEIPALApp\ Binding Type: http IP Address : All Unassigned Port : 5001 Host Name : (your hostname or leave blank) # Click OK → Start the site
✅
Test by visiting http://localhost:5001 from the server browser. The application should load.

🔒Configure SSL / HTTPS (Port 443)

For HTTPS access, you need a hostname and SSL certificate bundle from your client.

# In IIS Manager → select your site → Bindings # Add → Type: https, Port: 443 # SSL Certificate: select the certificate provided by client # To import a .pfx certificate: certlm.msc # Open Certificate Manager # Personal → Certificates → Right-click → All Tasks → Import # Browse to your .pfx file, enter password, import
4

Firewall & Network Configuration

PHASE 04 / 06

🔥Open Port 5001 in Windows Firewall

# Run as Administrator in PowerShell: New-NetFirewallRule ` -DisplayName "CEIPAL App Port 5001" ` -Direction Inbound ` -Protocol TCP ` -LocalPort 5001 ` -Action Allow New-NetFirewallRule ` -DisplayName "HTTP Port 80" ` -Direction Inbound ` -Protocol TCP ` -LocalPort 80 ` -Action Allow New-NetFirewallRule ` -DisplayName "HTTPS Port 443" ` -Direction Inbound ` -Protocol TCP ` -LocalPort 443 ` -Action Allow
⚠️
Also whitelist port 5001 on your router/hardware firewall. Login to your router admin panel and add a port-forwarding rule for port 5001 → server internal IP (192.168.1.100).

🌍Whitelist Outbound URLs

Your server must be able to reach these domains outbound. Add them to your firewall/proxy allowlist:

DomainPurpose
*.naukri.comAll Naukri subdomains
resdex.naukri.comResume database access
*.ceipal.comAll CEIPAL subdomains
# Test connectivity from server PowerShell: Test-NetConnection -ComputerName naukri.com -Port 443 Test-NetConnection -ComputerName ceipal.com -Port 443 # TcpTestSucceeded should show: True
5

Enable Shared Access from Office PCs

PHASE 05 / 06
This lets all recruiters in your office access the application from their own computers.

🗺️How Office Access Works

[ Recruiter PC 1 ]──┐ [ Recruiter PC 2 ]──┤──▶ [ Office Router / Switch ]──▶ [ Your Server 192.168.1.100 ] [ Recruiter PC 3 ]──┘ Internal LAN (port 5001) Running IIS + App │ ▼ [ Internet ] ──▶ naukri.com / ceipal.com

🔢Find Your Server's Local IP

# On the server, open Command Prompt: ipconfig # Note the "IPv4 Address" e.g. 192.168.1.100 # All recruiters will use this IP to access the app

💻Access from Recruiter PCs

Each recruiter opens their Chrome browser and navigates to:

# HTTP access (port 5001): http://192.168.1.100:5001 # OR using hostname (if configured): http://yourserver.company.com:5001 # HTTPS (if SSL configured): https://yourserver.company.com
💡
You can create a Windows Hosts file entry on each PC for a friendly name.
Open C:\Windows\System32\drivers\etc\hosts as Admin and add:
192.168.1.100   ceipal-server
Then access via http://ceipal-server:5001

🔑Set IIS Application Pool Permissions

Allow the app pool to read/write the D:\ drive folder.

# Right-click D:\CEIPALApp → Properties → Security # Click Edit → Add → type: IIS AppPool\CEIPALApp # Grant: Read, Write, Modify → OK
6

Final Checks & Go-Live

PHASE 06 / 06

✅Pre-Launch Checklist

  • Windows OS installed and activated
  • .NET Framework 4.5.2+ installed and verified
  • IIS 7+ installed and running (test http://localhost)
  • IIS Manager 8+ installed
  • Application files deployed to D:\CEIPALApp\
  • IIS Site created on Port 5001
  • SSL certificate imported (for HTTPS/Port 443)
  • Port 5001 opened in Windows Firewall
  • Port 5001 forwarded on router
  • Outbound URLs whitelisted (*.naukri.com, *.ceipal.com)
  • Static local IP set (e.g. 192.168.1.100)
  • Tested from server browser: http://localhost:5001
  • Tested from recruiter PC: http://192.168.1.100:5001
  • Chrome extension installed on all recruiter PCs
  • Recruiters warned NOT to use Naukri logins manually

⚠️
Critical Reminder: Once the server is configured, do not use Naukri logins manually on this server. The Naukri API will stop working if a manual login is detected on the same account. All access should go through the CEIPAL application only.
