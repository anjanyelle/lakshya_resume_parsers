2) Required Database Models (SQLAlchemy 2.0)

Candidates
- id (UUID, PK)
- email (unique, indexed)
- full_name
- phone
- location
- linkedin_url
- github_url
- summary
- years_experience
- current_title
- current_company
- status (pending, processing, success, failed)
- created_at, updated_at

WorkHistory
- id (UUID, PK)
- candidate_id (FK -> candidates.id, indexed)
- company_name
- job_title
- start_date
- end_date
- is_current
- location
- description
- display_order

Education
- id (UUID, PK)
- candidate_id (FK -> candidates.id, indexed)
- institution
- degree
- field_of_study
- start_date
- end_date
- gpa
- description

Skills
- id (UUID, PK)
- name (unique, indexed)
- category
- normalized_name (indexed)

CandidateSkills
- candidate_id (FK -> candidates.id, PK)
- skill_id (FK -> skills.id, PK)
- proficiency_level (beginner, intermediate, advanced, expert)
- years_experience

ParsingJobs
- id (UUID, PK)
- candidate_id (FK -> candidates.id, indexed)
- filename
- file_path
- status (queued, processing, success, failed)
- raw_text (Text)
- parsed_data (JSONB)
- confidence_score (Float)
- error_message (Text)
- started_at
- completed_at

Certifications
- id (UUID, PK)
- candidate_id (FK -> candidates.id, indexed)
- name
- issuing_organization
- issue_date
- expiry_date
- credential_id

3) Indexes and Constraints
- candidates.email unique + index
- skills.name unique + index
- skills.normalized_name index
- work_history.candidate_id index
- education.candidate_id index
- parsing_jobs.candidate_id index
- candidate_skills composite PK (candidate_id, skill_id)
- foreign keys with cascade delete on candidate

4) Alembic Migrations
- alembic.ini + env.py configured for SQLAlchemy 2.0
- initial migration creates enums and tables
- checkfirst=True for enums

7) Core Pipeline (Parsing Flow)
1. Upload file -> store in STORAGE_DIR
2. Create ParsingJob with status=queued
3. Worker pulls job -> status=processing
4. Extract text (pdf/docx/OCR)
5. Parse entities (name, contact, work, education)
6. Normalize + map skills
7. Score confidence
8. Save to Candidate + related tables
9. ParsingJob status=success or failed

8) Required API Endpoints
- POST /api/v1/upload
- GET /api/v1/candidates
- GET /api/v1/candidates/{id}
- GET /api/v1/candidates/{id}/status
- POST /api/v1/candidates/{id}/reprocess
- DELETE /api/v1/candidates/{id}
